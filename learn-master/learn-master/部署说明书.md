### 1.1 部署全景

### 3.2 Thinking 推理模式配置详解

系统通过 `AsyncLocalStorage` + 自定义 fetch wrapper 实现 **Thinking/推理模式的线程安全透明注入**。这是系统的一大技术创新：

```
┌─────────────────────────────────────────────────────────────────┐
│           Thinking 透明注入机制 — 部署配置详解                     │
│                                                                  │
│  全局开关:                                                       │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ LLM_THINKING_DISABLED=true   # 全局禁用 thinking (默认)   │   │
│  │ # 设为 false 后，各请求可独立控制 thinking 开关             │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
│  运行时数据流:                                                    │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                                                          │   │
│  │  API 请求到达                                              │   │
│  │      │                                                   │   │
│  │      ▼                                                   │   │
│  │  thinkingContext.run(config, () => {                     │   │
│  │      │                                                   │   │
│  │      ▼                                                   │   │
│  │  callLLM(model, prompt, source, thinking)                 │   │
│  │      │                                                   │   │
│  │      ▼                                                   │   │
│  │  AI SDK generateText / streamText                        │   │
│  │      │                                                   │   │
│  │      ▼                                                   │   │
│  │  自定义 fetch wrapper 拦截 HTTP 请求                       │   │
│  │      │                                                   │   │
│  │      ├── globalThis.__thinkingContext.getStore()          │   │
│  │      │   → 读取当前请求的 ThinkingConfig                  │   │
│  │      │                                                   │   │
│  │      ├── OpenAI 原生: 注入 body.reasoning                 │   │
│  │      ├── Anthropic 原生: 注入 thinking blocks             │   │
│  │      ├── Kimi/DeepSeek/GLM: {thinking: {type}}           │   │
│  │      └── Qwen/SiliconFlow: {enable_thinking: bool}       │   │
│  │      │                                                   │   │
│  │      ▼                                                   │   │
│  │  发送到提供商 API                                          │   │
│  │                                                          │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
│  各提供商 Thinking 参数对照:                                      │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ 提供商        Thinking 启用            Thinking 禁用       │   │
│  │ ──────────────────────────────────────────────────────── │   │
│  │ OpenAI       reasoning: {effort}      reasoning: undef   │   │
│  │ Anthropic    thinking: {type, budget} thinking: undef    │   │
│  │ Kimi         thinking:{type:'enabled'} thinking:{type:   │   │
│  │                                          'disabled'}    │   │
│  │ DeepSeek     thinking:{type:'enabled'} thinking:{type:   │   │
│  │                                          'disabled'}    │   │
│  │ GLM          thinking:{type:'enabled'} thinking:{type:   │   │
│  │                                          'disabled'}    │   │
│  │ Qwen         enable_thinking:true       enable_thinking: │   │
│  │                                          false           │   │
│  │ SiliconFlow  enable_thinking:true       enable_thinking: │   │
│  │                                          false           │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 API Key 服务端加密存储

用户的第三方 API Key 通过 **AES-256 加密存储在 PostgreSQL DtApiKey 表**中，密钥永不离开服务端：

```
┌─────────────────────────────────────────────────────────────────┐
│              API Key 安全存储架构                                 │
│                                                                  │
│  浏览器                         服务端                           │
│  ────────                      ────────                         │
│                                                          │
│  用户输入 API Key                                            │
│      │                                                      │
│      ▼                                                      │
│  POST /api/v1/settings                                      │
│  { apiKey: "sk-xxx" }                                      │
│      │                                                      │
│      └──────────────────────────────►  AES-256 加密          │
│                                          │                   │
│                                          ▼                   │
│                                      INSERT INTO dt_api_keys │
│                                      (api_key = "encrypted") │
│                                                                  │
│  LLM 调用时:                                                    │
│      │                                                      │
│      ▼                                                      │
│  POST /api/v1/turns                                         │
│      │                                                      │
│      └──────────────────────────────►  从 DB 读取            │
│                                          │                   │
│                                          ▼                   │
│                                      AES-256 解密            │
│                                          │                   │
│                                          ▼                   │
│                                      注入 AI SDK 请求        │
│                                          │                   │
│                                          ▼                   │
│                                      → OpenAI API (HTTPS)   │
│                                                                  │
│  前端永远无法读取明文 API Key                                     │
│  Prisma DtApiKey 模型: userId + provider 联合唯一索引              │
└─────────────────────────────────────────────────────────────────┘
```

## 四、快速部署指南

### 4.1 Docker Compose 一键部署 (推荐)

```bash
# Step 1: 克隆项目
cd smartlearn

# Step 2: 配置环境变量
cp .env.example .env
# 编辑 .env，至少填入一个 LLM API Key
# 推荐: OPENAI_API_KEY=sk-your-key
# 国内: SPARK_API_KEY=your-key + SPARK_API_SECRET=your-secret

# Step 3: 一键启动
docker compose up -d

# Step 4: 验证部署
curl http://localhost:3000/api/v1/health
# 预期响应: {"success":true,"data":{"version":"2.0.0","capabilities":[...]}}

# Step 5: 打开浏览器
# http://localhost:3000
```

### 4.2 本地开发环境部署

```bash
# Step 1: 安装依赖
nvm use          # Node.js 22 (读取 .nvmrc)
npm install

# Step 2: 启动 PostgreSQL (需要本地安装)
# 或使用 Docker 单独启动数据库:
docker run -d --name smartlearn-db \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=smartlearn \
  -p 5432:5432 \
  postgres:16-alpine

# Step 3: 初始化 pgvector 扩展
docker exec smartlearn-db psql -U postgres -d smartlearn \
  -c "CREATE EXTENSION IF NOT EXISTS vector;"

# Step 4: 配置 .env
cp .env.example .env
# DATABASE_URL=postgresql://postgres:postgres@localhost:5432/smartlearn

# Step 5: 数据库迁移
npx prisma migrate dev

# Step 6: 启动开发服务器
npm run dev
# → http://localhost:3000 (Turbopack 热更新)
```

### 5.1 Dockerfile 多阶段构建详解

SmartLearn 使用三阶段 Docker 构建，最小化生产镜像体积并实现安全加固：

```
┌─────────────────────────────────────────────────────────────────┐
│              三阶段 Docker 构建流程                               │
│                                                                  │
│  Stage 1: deps (依赖层)                                          │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ FROM node:22-alpine                                      │   │
```
