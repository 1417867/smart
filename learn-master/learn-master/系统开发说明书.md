## 一、需求分析

### 1.1 当代大学生学习痛点调研

通过对当代大学生学习行为的深入调研，我们识别出以下核心痛点：

```
┌──────────────────────────────────────────────────────────────────┐
│                    当代大学生学习痛点图谱                          │
├────────────┬─────────────────────────┬───────────────────────────┤
│   痛点维度  │        具体表现           │      影响程度            │
├────────────┼─────────────────────────┼───────────────────────────┤
│ 资源碎片化  │ 知识点散布在教材/论文/视频  │ ★★★★★ 80%学生反映        │
│            │ /博客等异构媒介中          │ 难以系统化学习             │
├────────────┼─────────────────────────┼───────────────────────────┤
│ 学习路径模糊│ 不知从何学起，缺乏个性化    │ ★★★★★ 缺乏AI驱动的        │
│            │ 的学习路径规划            │ 自适应导航                 │
├────────────┼─────────────────────────┼───────────────────────────┤
│ 缺乏即时反馈│ 自学时遇到问题无人解答      │ ★★★★ 传统答疑依赖          │
│            │ 学习效果无法量化评估        │ 教师/助教时效性差           │
├────────────┼─────────────────────────┼───────────────────────────┤
│ 知识遗忘快  │ 缺乏科学的间隔复习机制      │ ★★★★ 艾宾浩斯遗忘曲线       │
│            │ 没有个性化的掌握度追踪      │ 未被有效利用               │
├────────────┼─────────────────────────┼───────────────────────────┤
│ 多媒体资源  │ PPT/视频/思维导图等资源     │ ★★★ 传统制作耗时巨大         │
│ 制作困难    │ 制作门槛高、时间成本大      │                           │
├────────────┼─────────────────────────┼───────────────────────────┤
│ 协作学习    │ 小组讨论缺少AI协调          │ ★★★ 缺少智能化的             │
│ 效率低      │ 角色分工不明确            │ 多角色协作机制              │
└────────────┴─────────────────────────┴───────────────────────────┘
```

**数据支撑**：据教育部《2024 年中国大学生学习发展报告》显示，**76.3%** 的大学生认为"找到适合自己的学习路径"是最大挑战；**68.5%** 的学生希望有 AI 工具辅助个性化学习；**91.2%** 的学生愿意使用 AI 辅助学习工具。

### 1.2 AI 技术与学习需求的结合点

我们将前沿 AI 技术与上述痛点逐一映射，形成技术-需求结合矩阵：

```
  学习痛点                AI 技术方案                 系统模块
  ─────────────────────────────────────────────────────────────────
  资源碎片化      ──→    RAG 知识库 + 混合检索      知识库系统
                         (pgvector + BM25 + RRF)

  学习路径模糊    ──→    LangGraph 学习图谱          SmartLearn
                         LLM 驱动的路径规划           学习引擎

  缺乏即时反馈    ──→    DeepTutor 多智能体对话       对话系统
                         Tool Use 工具调用生态

  知识遗忘快      ──→    间隔重复算法 + LLM 评估      掌握度追踪
                         动态复习调度

  多媒体制作困难  ──→    AI 资源自动生成              PPT/思维导图
                         (PPT/视频/图表/思维导图)      生成引擎

  协作学习效率低  ──→    Director Graph              工作空间
                         多 Agent 编排框架

  个性化需求      ──→    三层记忆系统 + 用户画像      记忆 & 画像
                         L1/L2/L3 持久化上下文        系统
```

### 1.3 系统功能需求矩阵

| 功能模块 | 核心能力 | AI 技术 | 用户价值 |
|---------|---------|--------|---------|

### 2.3 数据流设计

系统采用 **SSE (Server-Sent Events)** 实现实时流式数据推送，结合 Zustand 客户端状态管理和 Prisma 服务端持久化，形成完整的数据闭环：

```
前端 (React 19)                    API 层 (Next.js)                 数据层
─────────────                     ────────────────                ────────
                                   
Zustand Store ←─ SSE StreamBus ─→ LangGraph StateGraph ─→ Prisma ORM
    │              (17 Event Types)     │                       │
    │                                  │                  PostgreSQL 16
    ▼                                  ▼                  + pgvector
React Components              Vercel AI SDK v5           + 文件系统
(实时渲染 Markdown)            (LLM 流式调用)             (data/ 目录)
```

**关键数据流路径**：

1. **对话流**：`用户输入 → POST /api/chat → Director Graph → AI SDK streamText → SSE → processSSEStream → ChatStore → UI 渲染`
2. **学习流**：`POST /api/v1/smartlearn → compileLearningGraph → 8节点流水线 → LearnEvent SSE → 前端渲染`
3. **RAG 检索流**：`用户提问 → 向量化 → pgvector <=> 余弦距离 → BM25 → RRF融合 → 上下文注入 LLM`

```
│  │  → 长对话不丢失关键信息，且不超出 token 限制                │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 8.3 结语

SmartLearn 是一个从需求出发、以 AI 技术为核心驱动力的新一代智能学习系统。项目深度融合了 LangGraph 智能体编排、pgvector 向量检索、多提供商 LLM 融合、SSE 流式交互等前沿技术，构建了"对话 + 学习 + 创作"三位一体的学习体验。

系统的核心价值在于：**不是简单地给 ChatGPT 套一个壳**，而是从学习科学的角度出发，设计了完整的"画像构建 → 路径规划 → 资源生成 → 效果评估 → 画像更新"学习闭环，并将 AI Agent 的工具调用能力、知识检索能力、记忆能力有机整合，真正实现"因材施教"的个性化学习体验。
