## 一、设计总览

### 1.1 设计理念

SmartLearn 的设计遵循三个核心原则：

```
┌─────────────────────────────────────────────────────────────────┐
│                    SmartLearn 设计三原则                          │
│                                                                  │
│   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐          │
│   │   AI 原生    │   │  自适应闭环  │   │  多模态融合  │          │
│   │             │   │             │   │             │          │
│   │ 不是传统应用 │   │ 画像→规划    │   │ 文本+图片    │          │
│   │ + AI 聊天   │   │ →生成→评估   │   │ +视频+语音   │          │
│   │             │   │ →更新→再规划  │   │ +PPT+图表    │          │
│   │ AI 是核心   │   │             │   │             │          │
│   │ 不是外挂    │   │ 无限迭代优化  │   │ 全感官学习   │          │
│   └─────────────┘   └─────────────┘   └─────────────┘          │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 系统定位

SmartLearn 区别于传统"ChatGPT 套壳"产品，是一套**以学习科学为理论基础、以多智能体协作架构为技术核心**的新一代 AI 学习系统。系统内部署了 12 个 AI Capability（智能体能力）和 25+ Tools（工具），通过双引擎协同驱动，实现从"用户画像构建"到"个性化学习路径规划"到"AI 资源自动生成"到"学习效果评估"到"画像动态更新"的完整闭环。

### 6.2 创新点详细说明

**创新 1: 双引擎协同架构** — 传统 AI 学习工具通常只有单一的对话或推荐模式。SmartLearn 首创双引擎协同：DeepTutor 处理自由对话（11 种能力 + 25+ 工具），SmartLearn 学习图谱处理结构化学习流程（8 节点闭环），两引擎共享底层 RAG/Memory/Search/Tools 服务。

**创新 2: 三层记忆持久化** — L1 事件追踪 (JSONL 追加)、L2 语义摘要 (LLM 定期整合)、L3 知识合成 (跨会话 4 槽位: recent/profile/scope/preferences)。系统越用越"懂"用户。

**创新 3: 多提供商 Thinking 透明注入** — 通过 AsyncLocalStorage + 自定义 fetch wrapper，在 HTTP 请求层透明注入各提供商的 Thinking/推理参数，支持 OpenAI/Anthropic 原生 + Kimi/DeepSeek/GLM/Qwen 等国产模型的差异化 thinking 参数。

**创新 4: 学习闭环自适应** — 画像 → 路径规划 → 资源生成 → 效果评估 → 画像更新 → 新一轮规划。8 个维度 × 3 个档位，LLM 驱动的动态分析，真正的"因材施教"。

**创新 5: 渐进式技能加载** — 核心指令总是在每次对话中注入，按需技能通过 `read_skill` 工具动态加载，XML 摘要先注入减少 token 消耗。有限 context window 内支持无限技能扩展。

**创新 6: Context Window 智能管理** — 使用率 > 90% 自动触发裁剪，老旧 tool results 替换为占位符，多轮对话自动压缩为摘要。保证长对话不丢失关键上下文。
